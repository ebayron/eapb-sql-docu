Alerts and Automations

Alerts

  • Indexes
    • Unused Indexes - Check if it needs to be dropped.
    • Disabled Indexes - Either this needs to be dropped or forgot to be re-enabled.
  • Etc
    • Last success run of any job (database backup, index, etc) - Monitoring last success run can cover failed, disabled, and long running jobs.
    • Database that does not have Page Verify set to CHECKSUM - Lowers chance of corruption.
    • Always On secondary node is not synchronizing for the past n minutes - Proactively monitor AG health.

Maintenance

  • Backup with DBCC CHECKDB. If database is too large, DBCC CHECKDB can be on a separate job and broken into stages:
    • DBCC CHECKALLOC (ex: Monday)
    • DBCC CATALOG (ex: Tuesday)
    • DBCC TABLE (ex: scattered from Wednesday to Sunday)
  • Perform DIFF backups with CHECKSUM, if it finds a corruption, backup will fail. It is better to know this early than after restoration.
  • Rebuild and Reorganize Indexes
