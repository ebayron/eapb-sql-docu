Troubleshoot SQL Server Database Engine intermittently cannot connect to SSRS

Problem: SQL Server Database Engine intermittently cannot connect to SSRS.

Error log shows:

  • SQL Server Error 258, Shared Memoy Provider. Timeout Error
  • SQL Server Error 258, Unable to complete login process due to delay in login response

Troubleshooting:

  • Generic connection failed found in SQL Server logs, SSRS logs, and Event Viewer logs.
  • Database Log Backup has also failed during that time.
  • Historical logs of CPU and Memory shows spike during that time.

Recommendation:

  • Check with NOC if there are any activies performed since the problem started.
  • Check with the job owner, if applicable, if large data are being processed during these times.
  • Check with the DB server owner if it is recommended to increase memory/CPU (whichever spiked).
