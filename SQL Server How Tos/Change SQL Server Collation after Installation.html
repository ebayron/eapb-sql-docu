Changing SQL Server Collation After Installation
  1. Note the collation needs to be changed.

  2. Note the installation path of SQL Server (ex: D:\Program Files\Microsoft SQL Server\MSSQL13.MSSQLSERVER\MSSQL\Binn).

  3. If there are user-databases that are already has the correct collation, de-attached them first (possible to encounter error 1934 and 3434).

  4. Stop SQL Server services.

  5. Run command prompt as administrator and change directory to the installation path.

  6. Type the following command: sqlservr -m -T4022 -T3659 -s"SQLEXP2014" -q"SQL_Latin1_General_CP1_CI_AI" (-s is only for named-instance servers).

  7. Start the service once done.

  8. Check database server (attach the de-attached databases, if any) and each database (sys.databases) for the collation.
